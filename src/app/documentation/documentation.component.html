Getting started

To use bimsync Manager, you must be connected to bimsync using your usual credentials.

Click on "Connect" or "Getting Started"

Then grant the application the permission to access to your bimsync account.

You can now see the list of your bimsync projects.

Furthermore, to allow bimsync Manager to create issue boards on your project, you must authorized it first. Just click on authorise.

Then grant again the application the permission to access to your bimsync account.

Create a new bimsync project

bimsync Manager allows you to to create a new bimsnc project based on a template. This template is a .json configuration text, where you describe how you want to create your project.

To create a new bimsync project with bimsync Manager :

  • Click on "Create Project
  • Type in your configuration text

This configuration text contain a series of projects to be created, enclosed between "{{ '{' }}""{{ '}' }}" and separated by comma. Each project contain a series of properties :

projectName

This is the name of your new project.

projectDescription

This is the description of your project. This property is optional.

Users

A list of members to be added to the new projet. This list is optionnal. The user must already a registered bimsync user, you can’t invite new users.

Each user must have the following format :

  • id: The ID of the user. This ID can be found in the URL of his profile. Just look for the last part of the URL, after “/members/“.
  • role: The role of the user added to the project, it can be “administrator” or “member”

Models

A list of model to be created in the new projet. This list is optionnal.

Each model must have the following format :

  • name: The name of the model to be added to your project

Board

A list of new issue boards to be created in your new project. This list is optionnal. Each new issue board will be created as a BCF 2.0 board. You can’t delete the "BCF" default issue board, it must be manually archived.
Boards follows this schema :

Name

This is the name of your new board

Statuses

A list of status to be added to your board.  This list is optionnal. If you add any new status, bimsync Manager will delete all existing ones and create yours. If you don’t add any status, bimsync Manager will keep the existing ones (“Open” and “Closed”). Furthermore, you must always have a status with the “open” type. If you don’t add any, bimsync Manager will keep the existing one.

Statuses follow this schema :

  • Name : The name of the status
  • Color : The Hex colour for the status (http://www.color-hex.com/). The following color code are used by bimsync :
  • Type : the type of the status. It can be “open”, “candidate” or “closed”.

Types

A list of types to be added to your board.  This list is optionnal. If you add any new type, bimsync Manager will delete all existing ones and create yours. If you don’t add any type, bimsync Manager will keep the existing ones (“Error”, “Warning”, “Info” and “Unknown”).

Types follow this schema:

  • Name : The name of the type
  • Color : The Hex colour for the type (http://www.color-hex.com/).

Power BI dashboard

Bimsync Manager allows you to connect your bimsync account to Power BI.

Set Up

Download one of the Power BI template below and open it.

To connect the Power BI template to you bimsync account, click on "Edit Parameter" and add your Power BI secret code.

Then click on "Apply Changes" to load your data into Power BI.

Power BI can take some time to be retrieve all the data, especially if you have access to a lot of projects.

If asked, all permissions must be set to Anonymous. Privacy levels must be set to "Public".

If you get stuck with permission issues, clear all permissions in "Data sources settings" and hit "Refresh".

Project Dashboard

The Project Dashboard template allows you to extract data from a specific project. This enable data extraction from any models. To use this dashboard, you must add your bimsync project Id to the Power BI parameters. This Id is available in the project list of bimsync Manager.

To be able to extract data from your model, the following tables are available:

  • AllRevisions : A list of the revisions in the project
  • Groups : A list of IfcGroups, by revision. This include objects likes IfcZones, IfcBuildingSystem, ...  http://www.buildingsmart-tech.org/ifc/IFC4/final/html/schema/ifckernel/lexical/ifcgroup.htm
  • IfcBuildingStorey : A list of storey, by revisions
  • IfcDoor : A list of doors, by revision
  • IfcFurnishingElement : A list of furniture, by revision
  • IfcSpace : A list of spaces, by revision
  • IfcWindow : A list of windows, by revision
  • LoadedRevision : A list of loaded revisions. For performance reason, data is only extracted for the 3 last revisons of every model. If you need more revisions, you need to edit manually the Power BI queries.
  • Models : A list of all models for the current project
  • Project : Informations about the loaded project
  • Relation : A list of all IfcRelationship, by revisions. You can use this table to link objects between tables.

Global dashboard

The  Global Dashboard template give you an overview of all your binsync projects, along with their members, models, revisions, ...

It can be used for cross-project analysis and management. However, for performance reason, you can’t use it extract data from all models of all your projects. To do so, use the Project Dashboard on a specific project.

The Global Dashboard template come with four sample pages, displaying generic information about your projects.
To find more, and extract meanifull relationship from your projects, the template contain the following tables :

  • Members : A list of relation between a project and a bimsync user
  • Models : All models, by project
  • OwnerHistory : The owner history of all the IFC models uploaded on bimsync
  • Projects : A list of all your bimsync projects
  • RefreshDate : A simple table to get the date of the last refresh
  • Revision : All revisions, by model and project
  • Users : A list of all bimsync users you are aware of

These tables are linked together with Power BI relations.

These templates are also set up to be uploaded on Power BI Online, where you can refresh them periodically without any intervention.
