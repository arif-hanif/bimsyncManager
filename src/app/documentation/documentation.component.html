Getting started

To use bimsync Manager, you must be connected to bimsync using your usual credentials. Click on Connect to grant the application access to your bimsync account. Furthermore, to allow bimsync Manager to create issue boards on your project, you must authorized it first. Just click on authorise.

Create a new bimsync project

bimsync Manager allows to to create a new bimsnc project based on a template. This template is a .json configuration text, where you describe how you want to create your project.

To create a new bimsync project :

  • Click on "Create Project”
  • Type in your configuration text
This configuration text contain a series of projects to be created, enclosed between and separated by comma. Each project contain a series of properties : //Code1 projectName : the name of your project projectEescription : the description of your project. This property is optional. Users: //An list of member to be added to the new projet. This list is optionnal. The user must already a registeted bimsync user, you can’t invite new users. Each user must have the following format : //Code2 User ID The ID of a given user can be found in the URL of his profile. Just look for the last part of the URL, after “/members/“. Role : The role of the user added to the project, it can be “administrator” or “member” Models : /A list of model to be created in the new projet. This list is optionnal. Each user must have the following format : //Code3 Board : //A list of new issue boards to be created in your new project. This list is optionnal. Each new issue board will be created as a BCF 2.0 board. You can’t delete the "BCF" default issue board, it must be manually archived. Boards follows this schema : //Code4 Name : the name of your new board Statuses : A list of status to be added to your board. This list is optionnal. If you add any new status, bimsync Manager will delete all existing ones and create yours. If you don’t add any status, bimsync Manager will keep the existing ones (“Open” and “Closed”). Furthermore, you must always have a status with the “open” type. If you don’t add any, bimsync Manager will keep the existing one. Statuses follow this schema : //Code5 Name : The name of the status Color : The Hex colour for the status (http://www.color-hex.com/). The following color code are used by bimsync : Type : the type of the status. It can be “open”, “candidate” or “closed”. Types : A list of types to be added to your board. This list is optionnal. If you add any new type, bimsync Manager will delete all existing ones and create yours. If you don’t add any type, bimsync Manager will keep the existing ones (“Error”, “Warning”, “Info” and “Unknown”) Types follow this schema //Code6 Name : The name of the type Color : The Hex colour for the type (http://www.color-hex.com/).
